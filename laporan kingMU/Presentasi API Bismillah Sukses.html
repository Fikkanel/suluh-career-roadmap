Version control untuk database

Apa itu Laravel Migration?

Migration adalah version control untuk database: setiap lingkungan memiliki struktur yang identik karena schema ditulis sebagai file PHP yang bisa dijalankan ulang.

Konsisten antar anggota tim, staging, dan production.

Bisa rollback saat perubahan schema bermasalah.

Menjadi dokumentasi hidup untuk struktur database.

Migration File
php artisan migrate
Database Table
php artisan migrate php artisan migrate:rollback php artisan make:migration create_skills_table

Tabel identitas dan peran

Struktur Migration: Tabel users

// intended design snippet Schema::create('users', function (Blueprint $table) { $table->id(); $table->string('name'); $table->string('email')->unique(); $table->string('password'); $table->enum('role', [ 'admin', 'user', 'mentor', 'institution' ])->default('user'); $table->unsignedTinyInteger('age')->nullable(); $table->string('province')->nullable(); $table->string('education_level')->nullable(); $table->string('education_major')->nullable(); $table->json('personality_scores')->nullable(); $table->rememberToken(); $table->timestamps(); });

Highlight teknis

role enum memisahkan admin, user, mentor, dan institution sejak level data.

personality_scores disimpan sebagai JSON agar skor RIASEC dan Big Five fleksibel.

Kolom profil dasar mendukung Context Score tanpa mengambil data berlebihan.

Basic Auth: Gerbang Mendapatkan JWT

POST /api/v1/auth/login Authorization: Basic base64(email:password) if credentials valid: return { "token_type": "Bearer", "access_token": "eyJhbGciOi...", "expires_in": 3600 }
01
Encode

02
Header
Kirim Authorization: Basic.
03
Verify
Server cek kredensial.
04
Issue
JWT diterbitkan.
05
Use
Client pakai Bearer token.

Basic Auth bukan untuk akses resource. Ia hanya pintu pertama untuk mendapatkan JWT.

JWT: Struktur dan Cara Kerja

Header

alg: HS256
typ: JWT

Payload

user_id, role, email, iat, exp

Signature

HMACSHA256(header + payload + secret)

01
Login
Client menerima JWT setelah Basic Auth.
02
Bearer
Setiap request membawa Authorization: Bearer.
03
Middleware
ValidateJWT cek signature dan exp.
04
Controller
Jika valid, request lanjut ke controller.
05
Logout
Token di-blacklist di Redis.

API Key: Akses Data Publik

Endpoint yang memakai API Key

GET /api/v1/careers GET /api/v1/impact/public Header: X-API-Key: {key}

Dipakai untuk institusi mitra, peneliti, dan statistik publik tanpa membuka data individual pengguna.

01
Issue
Admin membuat key unik untuk mitra.
02
Header
Mitra mengirim X-API-Key.
03
Validate
Middleware cek key di database.
04
Audit
IP, endpoint, dan waktu dicatat.
05
Revoke
Satu key bisa dicabut tanpa ganggu key lain.
